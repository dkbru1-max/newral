Используй docs/PROJECT_BRIEF.md и docs/PORTAL.md для контекста. Цель: добавить на портал автоматическое обновление данных в реальном времени и подготовить новый графический интерфейс агента для Windows.

1) **Автообновление портала**: Реализуй механизм live-обновления данных на фронтенде (React) через WebSocket или Server-Sent Events (SSE).
   - Настрой серверную часть (например, в telemetry-service или соответствующих сервисах) на отправку обновлений раз в секунду (частота должна быть настраиваемой через конфиг) и мгновенно при ключевых событиях (триггерах).
   - Обеспечь обновление следующих блоков портала в реальном времени: список подключённых агентов, очередь задач/их статусы, показатели нагрузки (например, число задач в обработке, производительность) и глобальный режим AI.
   - В gateway (Nginx) при необходимости добавь поддержку проксирования SSE/WebSocket (для SSE достаточно корректных заголовков и флуш, для WebSocket – директивы upgrade).
   - На клиенте: подключайся к соответствующим /api/ потокам и обновляй состояние UI без перезагрузки страницы. Убедись, что при отключении/переподключении соединения данные восстанавливаются корректно.

2) **Минорные правки UI**: Приведи текстовые элементы портала в актуальный вид.
   - Удали в интерфейсе все упоминания слова «инвестор» – замени на «демо» или «сводка» в зависимости от контекста. Например, вкладку "Investor Demo" переименуй в "Demo" (или "Демо-проект"), раздел «для инвесторов» – в «Сводка» и т.д.
   - Обнови favicon портала на брендированный: возьми логотип Newral (`newral_big_logo.png`) и сгенерируй на его основе favicon (без слова Newral, только буква N) и набор иконок (PNG) разных размеров (например, 16x16, 32x32, 64x64). Добавь эти файлы в публичные ресурсы фронтенда (например, `frontend/public/`) и пропиши ссылки в HTML. Убедись, что favicon отображается во всех размерах (вкладки браузера, ярлыки).

3) **Графический интерфейс агента (Windows)**: Создай полноценное GUI-приложение для Windows-агента на Rust.
   - Выбери подходящий фреймворк для графического интерфейса (например, `egui`, `tauri`, `native-windows-gui` или `wails`). Реализуй простое окно приложения в едином стиле с веб-порталом Newral: используй логотип и фирменные цвета, оформляй элементы управления в общем дизайне.
   - **Умный лог:** В окне должен быть виджет логов, где выводятся события/сообщения агента. Сделай цветовую индикацию уровней логов: информационные (info) – один цвет, предупреждения (warn) – другой, ошибки (error) – красным, успешные операции (success) – зелёным. Добавь фильтры (checkbox или переключатели) для отображения только выбранных уровней логов.
   - Отображай статус агента: на видном месте индикация соединения с сервером (например, зелёная точка при подключении, серая/красная при отсутствии), текущая выполняемая задача (если есть) и последний полученный результат задачи. Эти поля должны обновляться динамически по мере работы агента.
   - Устрани мерцание консольных окон при запуске агентского GUI. Для бинарника на Rust добавь атрибут `#![windows_subsystem = "windows"]` (в Cargo `[[bin]]` секции) или используй соответствующий флаг, чтобы приложение запускалось без консольного окна. Если агент запускает подпроцессы (песочницу), позаботься о скрытии их консолей тоже.
   - **Резерв на будущее:** Спроектируй приложение с учётом возможной установки агента как Windows-сервиса. Например, раздели логику ядра агента и UI: основной функционал (подключение, выполнение задач) вынеси в отдельный поток или модуль, чтобы позже его можно было запускать как фонового сервисного процесса. Пока реализацию службы не делаешь, но код строится так, чтобы добавить эту возможность (например, через флаг запуска “--service” без UI).
