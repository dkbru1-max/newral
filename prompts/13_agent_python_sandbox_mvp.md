Используй docs/PROJECT_BRIEF.md. Цель: реализовать MVP-песочницу на агенте.

Сделай в client/agent:
1) SandboxRunner:
   - Выполнение задач через subprocess "python" (или python3).
   - Перед запуском создается tmp директория (workspace), туда кладутся input файлы задачи.
   - Ограничения MVP:
     - timeout (например 60 сек) — обязателен
     - ограничить размер workspace (например 16 ГБ как абсолютный лимит, на MVP можно 512МБ для теста) — мониторить размер папки и прерывать
     - ограничить размер stdout/stderr (например 5-20МБ) — чтобы агент не утонул в логах
     - снизить приоритет процесса (Linux: nice, Windows: ниже приоритета если возможно)
   - 50% CPU строго не гарантируй, но заложи “троттлинг” как будущий интерфейс:
     - мониторинг CPU и при превышении — пауза/kill (на MVP можно просто логировать).

2) Формат задачи:
   - Добавь тип задачи "python_script":
     payload содержит python-код (строка) или ссылку на скрипт, плюс входные данные.
   - Агент сохраняет код в workspace/task.py и запускает python task.py.

3) Безопасность:
   - Никаких системных путей из payload не принимать.
   - Все файлы только внутри workspace.
   - Логи и результаты — строго ограниченные по размеру.

4) Документация:
   - docs/SANDBOX_MVP.md: что гарантирует MVP, что НЕ гарантирует, план усиления.
